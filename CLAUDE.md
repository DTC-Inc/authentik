## Git Workflow

**Branching Model:**
* `development` ‚Äî default branch (HEAD), active work lands here
* `release` ‚Äî stable/production branch, merged from development when ready
* `enhancement/{name}` ‚Äî branched from development for new functionality
* `problem/{name}` ‚Äî branched from development for bug fixes and issue resolution
* No `main` or `master` branches

**GitHub Issues & Labels:**
* New functionality uses the **enhancement** label, not "feature"
* Configure repository labels accordingly
